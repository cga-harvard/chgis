B. Description

1. Abstract

Polygons for China's regional city systems. Region subclasses define Macroregional systems and subsystems.

2. Purpose

Spatial framework for macroregional analysis at 1:1,000,000 scale.

3. Procedures Used (overview)

Regional city systems were delineated through a central place analysis on paper maps printed at approximately 1:1,000,000 scale. System boundaries were determined based on the location of categorized central places on the transportation network. Boundaries traced on acetate were scanned, vectorized, and edited from multiple, irregular map sheets to form a single coverage. See II.3 below.

4. Sources and related Data Sets

Central places and transportation network georeferenced by ACASIAN, Griffiths University, Australia (q.v.).

Hierarchical status of central places from China-T Dataset, University of California, Davis.

Central place analysis procedures based on John Marshall (1989).

3. Processing steps

Objective: compile a new Arc/Info coverage of new regional city system boundaries drawn based on a new central place analysis. Steps 1-8 work on separate coverages for each macroregion or map frame. Steps 9-16 work on a merged all-China coverage. See the .log file for additional details.

  1. Plot "Central Places Analysis Maps" using Regmap.aml. The appearance of these maps is defined in $CHINA/cp/t-zhen.aml, option t2, which contains numbered latitude and longitude lines, city names and type, provincial and international borders, roads (superhighways, national highways, unpaved national highways, provincial highways, and local roads), railroads, rivers (major and minor), the original macroregional boundaries, and the original shaded regional city system polygons. The map frames are defined in $AML/getframe.aml, which specifies the name, extent, scale, and orientation for maps of macroregional subsystems and other areas. (The scale of the maps will vary depending on the density of cities and roads, but should be approximately 1:1,000,000.) For example, to plot the map of Region 5.2, use "&r regmap r52 zhen t2."
  2. Trace new regional city system and (where applicable) macroregional borders onto mylar or high quality tracing paper. Write the name of the map and the late in the lower right corner of each mylar. (G. William Skinner)
  3. Prepare mylar for scanning by marking tics at the intersections of longitude and latitude lines. Then number the tics with 5-digit number consisting of the latitude followed by the longitude. Use a pencil to go over any light markings or small gaps in the lines forming regional system borders. Check intersections and cities to make sure lines are not discontinuous. (Michele Ladenson w/ Mark Henderson).
  4. Scan each mylar, saving it into a .tif file labeled rXY or rX(Y)Z, where X is the macroregion number, Y is the subsystem, and Z is a compass point used to differentiate between coverages in the same macroregion or subsystem. Use imageline.aml to create a useable coverage of the same name. (Michele Ladenson, Mark Henderson, w/ Doug Messenger)
  5. Using the Editing.aml program, add the tics into the coverage, numbering them as marked. Add arcs in cases where the lines on the mylar were too light or thin for the scanner to pick them up and remove extraneous editing marks from saved coverage (select continuous, nonselect delete). Careful editing may be needed to remove small circles surrounding towns on the RCS borders generated by step 2 above. (Ladenson, programming by Messenger)
  6. Once coverages have been edited, georeference them by using registerchina.aml. (Henderson and Ladenson)
  7. Using either the Editing program or Arcedit commands and showing neighboring coverages as backcoverages, remove (select many, delete) overlapping or superceded lines at the edges of coverages, referring to the notes on the original mylars as needed. (Ladenson)
  8. Generate macroregion coverages (r#) from the multiple coverages for each macroregion using the ‘arcsnap’, ‘nodesnap’ and ‘put’ commands. Run addsource.aml to add a source code (the frame name) and set BTYPE = 3 (to denote a regional city system boundary). (Henderson and Ladenson)
  9. Begin working with an all-China coverage. Make a copy of $CHINA/hrs/rcs. Working one macroregion at a time, delete original arcs and replace them arcs with arcs from the macroregional coverage that has been created in steps 1-8. The new working version of the all-China coverage should be called rcs98new_1. If, during the editing process, it is necessary to print maps showing updated arcs, macros can be modified to refer to this new working version. (Henderson and Ladenson)
  10. Add offshore arcs so that coastal RCSes form closed polygons. Using $CHINA/hydro/coastline as a background and snap coverage, ensure that traced RCS arcs stop exactly at the coastline (these should be dangles at this point). Then add new arcs, extending arbitratily into the ocean and forming closed polygons. Code these new arcs BTYPE=2. Also check for arcs which cross major lakes as shown in $CHINA/hydro/polakes; split these arcs at the lake shores, and set the arc segments within the lakes to BTYPE=2.
  11. Begin the process of cleaning up the China-wide coverage. Run settol.aml to set tolerances, then clean the coverage to create polygon topology.
  12. Repair any dangles. In Arcedit, look for ordinary dangles (visible with DE NODE DANGLE) and cryptodangles (select arcs with LPOLY# = RPOLY#).
  13. Delete all of the labels in the coverage, then replace them with GET RCSCITY. (RCSCITY is a point coverage of regional cities produced by getallcp.aml.) Then build the coverage again, and check for label errors (visible with Arcplot's LABELERRORS command). No polygons should have more than one label, unless they define a regional city system with two "twin" regional cities; add arcs as necessary to separate other cities' systems. No polygons should be without labels; determine whether polygons lacking labels are slivers or other errors, and remove arcs as necessary. After making edits, BUILD the coverage again and inspect LABELERRORS again.
  14. Set the SYSTEM and COLOR polygon attributes using selpoly.aml. (Refer to previous version of the RCS coverage or the source manuscripts to determine these values for each polygon.)
  15. Run makeRCSregions.aml to create MR and SUB region subclasses (defining macroregions and subsystems) and code arcs along region boundaries (see BTYPE codes 2-7 below). Finally, replace the $CHINA/hrs/rcs coverage with the output from makeRCSregions.
  16. Print "Central Places Analysis RCS Checkplots" using "&r regmap <region 0..9> zhen t4." If changes are required, follow the procedures in steps 2-8, marking corrections or "fine-tuning" amendments on mylar and vectorize them to create an all-China coverage of arcs to be changed. Then use Arcedit commands to substitute the corrected arcs into the RCS coverage. Repeat steps 15-16 to verify the changes.

A. Arc Attribute Table (AAT)

BTYPE

Boundary Type

See RCS.ABTYPE

2 = Arbitrary Boundary under water (in ocean or major lakes)

3 = Regional City System Boundary

4 = RCS and Macroregional Subsystem Boundary

5 = RCS and Macroregional System Boundary

6 = System or Subsystem Boundary (but not RCS, because the regional city is Janus-faced, oriented to two macroregions). Applies only to Linchuan, Jiangxi.

7 = System or Subsystem Boundary and boundary of an ordinary RCS which has also has an extended territory beyond this line in the adjacent macroregion. Applies only along the border of the Mongolian Steppes subsystem.

50 = International Boundary (from Digital Chart of the World)

SOURCE

Source of Arc

Arcs vectorized at UC Davis: indicates year vectorized and acetate map sheet on which the arc was traced. Map sheet codes beginning with the letter "r" are from the original central place analysis (February-March 1998); codes beginning with the letter "f" are "finetuning" adjustments (April-May 1998).

Other arcs: indicates coverage from which arc was derived (applies primarily to international boundaries).

B. Polygon Attribute Table (PAT)

SYSTEM

Macroregional System+Subsystem

Two-digit macroregional system designation. (First digit is MR {0..9}; second digit is SUB.)

RCS-T1

Identifier

China-T Datafile ID number (T1) of the Regional City of each system (polygon). For systems with two Regional Cities, the northwesternmost city's T1 is used.

EXTENDED

Supplemental identifier

0 = ordinary RCS polygon

1 = polygon representing the extension of an RCS into the Mongolian Steppes subsystem. This treatment accommodates cityless areas in that subsystem whose modern transport patterns appear to be oriented towards cities in adjacent regions.

COLOR

Display color

Arbitrary color number {1..5} assigned such that adjacent polygons have different colors.

C. EXTENDED region subclass (.PATEXTENDED)

RCS-T1

Identifier

China-T Datafile ID number (T1) of the Regional City of each system (polygon). For systems with two Regional Cities, the northwesternmost city's T1 is used. Regions in this class encompass both the ordinary and extended portions of RCSes.

D. MR region subclass (.PATRCS)

RCS-T1

Identifier

China-T Datafile ID number (T1) of the Regional City of each system (polygon). For systems with two Regional Cities, the northwesternmost city's T1 is used. The extended portions of RCSes along the border of the Mongolian Steppes subsystem are excluded from this subclass.

EXTENDED

Supplemental identifier

0 = ordinary RCS polygon

1 = polygon representing the extension of an RCS into the Mongolian Steppes subsystem

E. SYSTEM region subclass (.PATSYSTEM)

SYSTEM

Macroregional Subsystems

Two-digit macroregional system designation. (First digit is MR {0..9}; second digit is SUB.)

F. MR region subclass (.PATMR)

MR

Macroregional System

One-digit macroregional system designation:

  1. Inner Asia
  2. Manchuria
  3. North China
  4. Northwest China
  5. Upper Yangzi
  6. Middle Yangzi
  7. Lower Yangzi
  8. Southeast Coast
  9. Lingnan
  10. Yungui
