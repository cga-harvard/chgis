B. Description

1. Abstract

A listing of records from the ChinaT Dataset, representing over 12,000 cities and towns in China, which have been assigned to Regional City Systems and Macroregions. ChinaT towns which have not been georeferenced (n=60" ) are assigned probabilistically. An accompanying list shows towns which have been georeferenced but which are not included in ChinaT (n=60" ).

2. Purpose

Results of a macroregional analysis at 1:1,000,000 scale for use as a spatial component in further analysis of demographic and economic data.

3. Procedures Used (overview)

GIS data for central places were verified against a listing from the ChinaT dataset, then used in an overlay analysis with a GIS file defining regional city systems and macroregions. See II.3 below.

4. Sources and related Data Sets

Central places georeferenced by ACASIAN, Griffiths University, Australia (q.v.).

China-T Dataset and Regional City Systems coverage produced at the University of California, Davis.

1. Intended area of coverage

2. Completeness

3. Processing steps

1. A listing of all entries in the ChinaT dataset (excluding t9=0 satellite towns) is exported from SAS as a .dbf database file and imported as an INFO table, ChinaT.Dat. This table includes the variables t1, t3, t5, t6, t9, t109, and t702. A RELATE is established based on the t1 variable.

2. GetAllCP.aml generates a coverage containing all central place points which have related entries in ChinaT.Dat:

&r getallcp cp-all chinat//t702 0 999

3. GetCPmismatch generates a coverage containing all central place points which do not have related entries in ChinaT.Dat, and summarizes the results of this process in NOTINSAS.DAT.

4. AssignRCS2T1.aml overlays matched central place points with the Regional City Systems coverage to produce the assignment of T1 to RCS-T1 (see the format below for RCST1.DAT).

5. NotInGIS.aml generates a list of records in ChinaT which lack georeferenced points, and attempts to assign these records to systems based on the assignments of other records in the same county. The results are summarized in NOTINGIS.DAT and RCST1.DAT.

A. RCST1.DAT (produced by AssignRCS2T1.aml using output from GetAllCP.aml)

T1

ChinaT Identifier

8-digit identifier for central places used in China-T Dataset. Based on China's National Standard (guobiao) codes for early 1990, with significant changes made at UCD for analytical purposes. Note: in this file, there is one entry for each combination of T1 and RCS-T1. Thus, the T1 identifiers for Janus-faced towns which are assigned to multiple regions appear more than once in this file.

RCS-T1

Regional City System Identifier

China-T Datafile ID number (T1) of the Regional City of each regional city system. For systems with two Regional Cities, the northwesternmost city's T1 is used.

EXTENDED

Supplemental RCS Identifier

Indicates whether the central place falls in an ordinary RCS or the extended portion of an extended RCS

0 = ordinary

1 = extended

SYSTEM

Macroregional Subsystem Identifier

Two-digit macroregional system designation. (First digit is MR {0..9}; second digit is SUB.)

FOUNDinGIS

GIS method for assigning town to systems

0 = this point is not georeferenced and falls in a county which is divided among systems; it was could not be assigned to a system

1 = this point is georeferenced and was assigned to a single system using GIS overlay analysis

2 = this point is georeferenced and was assigned to multiple systems using GIS overlay analysis (indicates a Janus-faced city)

3 = this point is not georeferenced but was assigned to the system to which the other towns in the same county were assigned

B. NotInGIS.DAT

T1

ChinaT Identifier

8-digit identifier for central places used in ChinaT Dataset. Based on China's National Standard (guobiao) codes for early 1990, with significant changes made at UCD for analytical purposes.

T3

Name

Place name from ChinaT Dataset.

C. NotInSAS.DAT (produced by GetCPmismatch.aml)

CITY_TOWN_CODE

ACASIAN Identifier

8-digit identifier used by ACASIAN. Based closely on guobiao codes for late 1990. Set to zero for points not georeferenced by ACASIAN.

NAME

Place name in Pinyin

Derived from ACASIAN's PINYIN_NAME attribute and verified against ChinaT Dataset's T3 field.

T1

China-T Identifier 

8-digit identifier used in ChinaT Dataset. Based on China's National Standard (guobiao) codes for early 1990, with significant changes made at UCD for analytical purposes. For points georeferenced by ACASIAN, T1 will be the same value as CITY_TOWN_CODE unless that value is used by a different town listed in ChinaT; in such cases, T1 will be set to zero.
