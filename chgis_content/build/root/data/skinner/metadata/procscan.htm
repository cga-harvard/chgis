Procedures for Scanning, Georeferencing, and Vectorization

1.  Scan a  manuscript after tracing to produce a TIFF image.

 
2. Convert TIFF image file to arcs
Arc: imageline
!Usage: IMAGELINE <image> {1st tile number} {last tile number}
arc: IMAGELINE tile 3 5
This will convert tile3.tif, tile4.tif, and tile5.tif to vector coverages tile3_1,  tile4_1, tile5_1. If you exclude the tile numbers, then this will convert just one image.
arc: IMAGELINE singleimage will convert singleimage.tif to a coverage named singleimage_1.

 
 
3. Modify tics; Add label points; Clean up dangles, extend and patch holes using the EDITING program
Arcedit: &station 9999
Arcedit: editing
!!Usage: EDITING <cover> {load step #} {save step #} {1st tile number} {last tile number}
Arcedit: EDITING tile 1 2 3 5
Using the output of step 2, this will edit the three coverages tile3_1, tile4_1, and tile5_1 and then save them as tile3_2, tile4_2, tile5_2.
arc: EDITING singleimage 1 2 will edit the coverage singleimage_1 and save it as singleimage_2
NOTE: STEP NUMBER represents the processing sequence number, which is preceeded by an underscore(_). Normally use a 1 for <load step number> to begin editing and add one to this number for the {save step number}.
 
4. Obtain  or create Tic coverage (<tic_coverage>)
The tic coverage must the proper projection and contain the tics that are marked on the manuscript sheet(s).
See UGIS chapter 7 for more information.
5. Georeference to create <cover(#)_3> .
6. APPEND, PUT, or GET coverages, if necessary.
Add your recent updates to previous work on a coverage, or combine seperate tiles into one coverage.  Note that this may have been completed already in step 5.
7. Fix gaps and overlaps where the tiles have been joined together using the EDITING program


Scanning
 

Prepare the Manuscript for Scanning
Write the date on the manuscript
With a pencil or very thin pen, retrace faint linework
Identify at least 4 registration points (tics) which also exist in the GIS
(If tics don't already exist in the GIS, create them now)
Mark the Tic-ID numbers on the manuscript

 
 
Scan the manuscript
Login on Rodin then move to the appropriate directory
Start CIS (rodin% cis)
Double click on FILE to set the path; type the filename.
If you are scanning multiple tiles, include a sequential tile number in the names, i.e.tile1.tif, tile2.tif, tile#.tif etc.(In these instructions, (#) indicates the optional tile number).
Set the Scanwidth
The scanned image should be in TIFF (.tif) format.
REVERSE the image by pushing the SPECIAL button then the appropriate toggle.
Scanning threshold should be approximately 105 - 128 (lower numbers for more sensitivity).
Insert manuscript into scanner and Scan.
Readjust threshold until the best results are obtained, then re-scan the manuscript.
return

Editing

 
Use the EDITING program to repair the newly created vector coverage.
 
Edit Tics
Delete all of the tics that were created by default.  Replace these with tics that have a coresponding TIC-ID in the Tic Coverage.
Edit Arcs
Add arc to bridge undesireable gaps.  Delete any unwanted arcs.  There are may tools available in this menu.  More documentation to be added.
Edit Labels
If this will be a polygon coverage, add labels with appropriate ID.  For example _add new item_ and call it COUNTY-ID.  Then chose COUNTY-ID item any time that you are editing the labels of this coverage.  The item attributes must match those of the coverages to which the current coverage will be added.
Arc Attributes
Use this to add and edit the ID's for a line coverage.  This is similar to edit labels, but useful for coverages such as roads and rivers.
return
 

Georeferencing

Method 1: TRANSFORM
create <new coverage> <tic_coverage>
transform <old coverage> <new coverage>
projectcopy cover <tic_coverage> cover <new_coverage>
(see UGIS chapter 7 for details.  Especially useful is the diagram on page 7-56)
Method 2:
arc: &r  registerchina
!Usage:   registerchina <Coverage prefix> <Input version#> <Output version#>
!Example: &r registerchina R23  2  3   ...looks for R23_2, saves as R23_3
Automatically finds the tic coverage and preforms the transformation for any work on G.W. Skinner's China coverages.
Method 3:
arc: transappend
!Usage TRANSAPPEND <cover> <load step #> <1st tile number>
<last tile number> <transform cover>
<output append coverage name> {logical expression}
arc: TRANSAPPEND tile 2 3 5 ticcover joined_cover
Tansforms and joins tile3_2, tile4_2, and tile5_2 into a single coverage.  'ticcover' is used for the transformation and must be in the appropriate projection and have TIC-IDs that match the TIC-IDs of the seperate tiles.
