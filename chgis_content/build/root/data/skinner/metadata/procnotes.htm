Principles

1. Data created for this project will be used by many different people over time. You will use data files created long before you joined this project, and others will use files you created long after you have left this project.

2. In general, GIS projects should budget 5% to 10% of their time to documentation of data and procedures. That means 3-6 minutes per hour, or 24-48 minutes per 8-hour day, or 2-4 hours per 40-hour week.

Recordkeeping when Scanning and Vectorizing

1. Mark the source map with the date scanned. Also mark the registration points (tics) along with the TIC-ID numbers used for registration with GIS data. This should be done before scanning so that the scanned image file will also include this information.

2. Add comments to the LOG file for all Arc/Info coverages and Grids that are created and every time they are edited. (Use the command LOG <geodataset> ADD.) Comments should specify who edited the file (using initials), what the file represents, and why the file was created or edited. (Note that Arc/Info automatically records when and how files are modified.) For example:

Enter comment: mgh/Region 4 macroregions scanned from 6/20/97 tracing
Enter comment: mgh/edited to match with China coastline

3. If a coverage is to be edited, attribute tables for the edited features (e.g., arcs or points) should have a SOURCE item (field) added so that the lineage and significance of each feature can be determined. Whenever an arc or point is manually edited, a notation should be made in its SOURCE field. If an arc or point is copied from a different coverage, the SOURCE field should indicate the name of the coverage from which the feature is copied. Common abbreviations in the SOURCE field include S (for Scanned or Screen-digitized) and D (for Digitizer) followed by the date, which can be checked against the coverage's LOG file. Metadata for Completed Coverages

When development work is complete, the UCD GIS Metadata Form should be completed for each coverage. Completed forms should then be linked to the GIS web pages.


Map Production

1. All maps should be stamped with a date and credit in this format:

©G. William Skinner
20 June 1997, Mark Henderson
UCD Visualization Lab
Data copyright owner
Date, Map Author
Computer System

Maps should also include standard cartographic elements: a title, key legend, and scale bar; map scale should also be expressed as a representative fraction. If a nonstandard map projection is used, map projection information and/or a north arrow should be included.

2. When sent to a plotter or printer, all maps should be logged in maplog.txt. (This is done automatically by the plotmap command.)

3. In most cases, maps should be generated by Theme macros. Descriptions of Theme macros, including a list of coverages drawn, should be added to the Support documentation.
